# Frontend

Current frontend:

```text
web/
```

Stack:

- Next.js 16 App Router
- React 19
- TypeScript
- Tailwind CSS v4
- Recharts
- Radix/shadcn-style local primitives
- Lucide React icons
- SWR for interactive client panels
- openapi-fetch and generated OpenAPI types

## Run

```bash
cd web
pnpm install
pnpm dev
```

Or from repo root:

```bash
make web
make demo
```

## Type Generation

Backend OpenAPI drives the frontend API types.

```bash
make types
```

Generated file:

```text
web/src/lib/api.gen.ts
```

Do not edit this file manually.

## Data Flow

Most read-only pages use Server Components and `serverFetch`.

Interactive panels use client components:

- simulator controls
- local recent decision state
- brief history
- sidebar/nav counts

## Current Pages

### `/`

Commercial workflow dashboard.

Shows:

- UK pulse
- rollup chips
- upcoming calls
- customer filter
- at-risk drawer
- market news

### `/decision/[sku]/[channel]`

Detailed decision page.

Tabs/views:

- diagnosis
- options
- simulate

Important chart:

- `ForecastChart`
- forecast line
- 80% band
- promo windows
- calendar markers
- tooltip with forecast/target context

### `/promos`

Compact promotion ROI table.

Current implementation does not include a promo-story card.

### `/brief`

Customer-call brief flow.

## Design Rules

- Keep the UI focused on commercial decisions, not model internals.
- Show hL first; show GBP as approximate secondary context.
- Avoid real customer/retailer names.
- Use chart annotations for promos/events, but keep the page readable.
- Do not imply the target is official budget; call it target/budget estimate where context matters.
